## 1. Identifikasi Bug/Masalah (Minimal 3 Bug)

### **Bug 1: Karakter Konflik Git di `README.md`**
- **Deskripsi Masalah**: Terdapat sisa penanda konflik git (`<<<<<<< HEAD`, `=======`, `>>>>>>>`) di file `README.md`.
- **Dampak**: Merusak visualisasi halaman dokumentasi pada Git repository hosting.
- **Lokasi Kode**: [README.md](file:///c:/laragon/www/LK-10/README.md)

### **Bug 2: Celah Keamanan Mass Assignment (Penggunaan `$request->all()`)**
- **Deskripsi Masalah**: Controller web dan API langsung menyimpan data mahasiswa ke database menggunakan `$request->all()` saat menyimpan (`store`) dan memperbarui (`update`).
- **Dampak**: Celah keamanan yang memungkinkan user mengirimkan field yang tidak valid atau field sensitif tambahan di luar kendali validasi.
- **Lokasi Kode**: 
  - `app/Http/Controllers/MahasiswaController.php` (method `store` dan `update`)
  - `app/Http/Controllers/Api/MahasiswaApiController.php` (method `store` dan `update`)

### **Bug 3: TypeError (HTTP 500) pada Route Parameters Non-Integer**
- **Deskripsi Masalah**: Parameter route dideklarasikan menggunakan type-hint `int $id` pada method controller. Jika user mengakses halaman menggunakan parameter non-integer (misal: `/mahasiswa/abc/edit`), PHP akan memicu `TypeError` dan server mengembalikan status **500 Internal Server Error**.
- **Dampak**: Respons server tidak bersih bagi pengguna dan berpotensi membebani log aplikasi dengan error tingkat sistem.
- **Lokasi Kode**: 
  - [MahasiswaController.php](file:///c:/laragon/www/LK-10/app/Http/Controllers/MahasiswaController.php) (method `edit`, `update`, `destroy`)
  - [MahasiswaApiController.php](file:///c:/laragon/www/LK-10/app/Http/Controllers/Api/MahasiswaApiController.php) (method `show`, `update`, `destroy`)

### **Bug 4: Ketiadaan Penampil Pesan Error di Layout Utama (`layouts/app.blade.php`)**
- **Deskripsi Masalah**: Layout utama hanya menampilkan alert `success` menggunakan session. Jika terjadi kesalahan/error umum (`session('error')`), tidak ada penanganan visual untuk menampilkannya ke pengguna.
- **Dampak**: Pengguna tidak mengetahui detail kegagalan operasi.
- **Lokasi Kode**: [app.blade.php](file:///c:/laragon/www/LK-10/resources/views/layouts/app.blade.php)

## 3. Refactoring Kode (3 Bagian)

### **Refactoring 1: Penyederhanaan Rute Login di `routes/web.php`**
- **Masalah Sebelum**: Terdapat closure identik yang diduplikasi tiga kali untuk rute `/`, `/auth/login`, dan `/login-page`.
- **Hasil Refactoring**: Membuat closure di dalam satu variabel `$loginController` dan membagikannya ke tiga rute tersebut.

#### **Sebelum Refactoring**
```php
Route::get('/', function () {
    if (Auth::check()) {
        return redirect()->route('mahasiswa.index');
    }
    return view('auth.login');
});

Route::get('/auth/login', function () {
    if (Auth::check()) {
        return redirect()->route('mahasiswa.index');
    }
    return view('auth.login');
})->name('login');

Route::get('/login-page', function () {
    if (Auth::check()) {
        return redirect()->route('mahasiswa.index');
    }
    return view('auth.login');
})->name('login-page');
```

#### **Sesudah Refactoring**
```php
$loginController = function () {
    if (Auth::check()) {
        return redirect()->route('mahasiswa.index');
    }
    return view('auth.login');
};

Route::get('/', $loginController);
Route::get('/auth/login', $loginController)->name('login');
Route::get('/login-page', $loginController)->name('login-page');
```

### **Refactoring 2: Ekstraksi Validasi ke Form Request (`StoreMahasiswaRequest` & `UpdateMahasiswaRequest`)**
- **Masalah Sebelum**: Aturan validasi dideklarasikan secara manual di controller web dan API secara terpisah, yang melanggar prinsip DRY (Don't Repeat Yourself).
- **Hasil Refactoring**: Mengekstrak logika ke `StoreMahasiswaRequest` dan `UpdateMahasiswaRequest` untuk digunakan kembali oleh Web dan API.

#### **File Baru: `app/Http/Requests/StoreMahasiswaRequest.php`**
```php
namespace App\Http\Requests;

use Illuminate\Contracts\Validation\Validator;
use Illuminate\Foundation\Http\FormRequest;
use Illuminate\Http\Exceptions\HttpResponseException;

class StoreMahasiswaRequest extends FormRequest
{
    public function authorize(): bool
    {
        return true;
    }

    public function rules(): array
    {
        return [
            'nim'      => 'required|string|max:20|unique:mahasiswas,nim',
            'nama'     => 'required|string|max:100',
            'email'    => 'required|email|unique:mahasiswas,email',
            'jurusan'  => 'required|string|max:100',
            'angkatan' => 'required|integer|min:2000|max:' . date('Y'),
        ];
    }

    protected function failedValidation(Validator $validator)
    {
        if ($this->expectsJson()) {
            throw new HttpResponseException(
                response()->json([
                    'success' => false,
                    'message' => 'Validasi gagal',
                    'errors' => $validator->errors()
                ], 422)
            );
        }
        parent::failedValidation($validator);
    }
}
```
